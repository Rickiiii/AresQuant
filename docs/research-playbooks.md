# AresQuant Research Playbooks

> 本文档定义 AresQuant 的 A 股投研工作流。目标不是让系统替用户“拍脑袋荐股”，而是让每次观察、加仓、止盈、风控都有数据依据、持仓上下文、反证条件和复盘记录。

## 1. 产品形态：AresQuant 到底是什么

AresQuant 未来应是一个面向 Ricki 的 **A 股个人投研与策略工作台**。

它由五层组成：

```text
数据中心 Data Center
  ↓
策略与因子 Strategy / Factor
  ↓
回测与验证 Backtest
  ↓
投研工作台 Research Center
  ↓
个人持仓助手 Portfolio Copilot
```

一句话描述：

> AresQuant 帮你把行情、持仓、基金暴露、主题强弱、因子信号、回测结果和每日复盘串起来，形成可解释、可追踪、可迭代的投资辅助系统。

它不是：

- 自动荐股神器
- 实盘自动交易系统
- 券商下单终端
- 大而全金融数据终端
- 黑箱 AI 选股系统

它应该是：

- 你的 A 股投研仪表盘
- 你的持仓体检工具
- 你的主题观察池
- 你的策略验证环境
- 你的 14:30 盘中复盘助手
- 你的投资 thesis 记录器

## 2. 未来怎么使用

## 2.1 每天盘前

用户打开 Dashboard 或调用 Research API，查看：

```text
今日市场准备
- 昨夜海外科技 / 黄金 / 港股 / 汇率是否影响当前持仓
- 今日 A 股重要事件、财报、政策、行业会议
- 当前组合主要风险暴露
- 今日重点观察主题：AI、机器人、物理 AI、通信设备、黄金、中证 1000、绿电等
- 今日操作预案：哪些适合继续观察，哪些触发加仓/减仓条件
```

系统输出不应只是新闻摘要，而应回答：

```text
今天我的持仓和观察池，最需要注意什么？
```

## 2.2 盘中 14:30

这是 AresQuant 的核心使用场景之一。

系统输出：

```text
14:30 盘中复盘
1. 市场温度：指数、涨跌家数、成交额、北向/主线情绪
2. 主题强弱：AI、机器人、通信设备、大科技、黄金、绿电、小盘等
3. 持仓表现：股票 + 基金暴露
4. 异常信号：放量、破位、涨跌停、主题退潮、风格切换
5. 策略/因子信号：动量、估值、波动、换手是否支持继续持有
6. 操作建议：继续持有 / 分批加仓 / 观望 / 止盈 / 风控
7. 反证条件：什么情况说明今天的判断错了
8. 明日关注：需要继续验证的点
```

核心问题：

```text
现在这个时间点，我该继续拿、加一点、观望，还是先收缩风险？
```

## 2.3 收盘后

系统生成复盘：

```text
收盘复盘
- 今日判断是否被验证
- 持仓 thesis 是否增强或削弱
- 哪些主题只是冲高回落，哪些是真强
- 哪些标的进入观察池
- 哪些标的需要移出观察池
- 明天的操作计划
```

## 2.4 周末

系统做组合级体检：

```text
周度 Portfolio Review
- 当前股票/基金/ETF/黄金/现金配置
- AI/机器人/通信设备/大科技/黄金/绿电/小盘暴露
- 收益来源与回撤来源
- 哪些持仓 thesis 仍然成立
- 哪些仓位过重/过轻
- 下周加仓或止盈优先级
```

## 3. AresQuant 能否辅助 Ricki

可以，但辅助方式必须清晰：

## 3.1 能辅助的事情

### 1. 持仓体检

系统可以回答：

```text
我现在持仓里哪些风险最大？
哪些是趋势还在但估值偏高？
哪些是跌了但 thesis 没坏？
哪些是主题退潮要警惕？
```

### 2. 加仓/减仓决策辅助

系统可以给出：

```text
适合继续加仓：趋势强、估值未极端、主题仍强、仓位不高
适合分批建仓：逻辑好但短期波动大
适合观望：方向不明确或缺少催化
适合止盈：短期过热、主题退潮、仓位过重
适合风控：跌破关键条件或 thesis 被证伪
```

### 3. 主题观察

重点跟踪 Ricki 关注的方向：

- 物理 AI
- 机器人
- AI 算力 / CPO / 通信设备
- 数字经济 / 大科技
- 黄金
- 中证 1000 / 小盘风格
- 绿电
- 恒生科技 / 纳指映射

### 4. 选股想法生成

结合多因子系统和主题标签，输出：

```text
候选标的
- 为什么入选
- 哪个因子贡献最大
- 当前估值是否拥挤
- 是否属于强主题
- 风险点是什么
- 是观察、分批建仓还是等待回调
```

### 5. 复盘纠错

系统会记录之前判断，后续对照市场结果：

```text
上次判断：AI 主题强但短期过热，建议不追高
今日结果：AI 冲高回落，判断有效
后续动作：等待回踩，保留观察池
```

这会让投资框架越来越稳定，而不是每天从零开始。

## 3.2 不能替代的事情

AresQuant 不能替代：

- 用户最终决策
- 真实交易执行
- 投资顾问资格
- 对突发政策/黑天鹅的完全判断
- 数据源异常时的人工核验

所有建议都必须保留：

```text
依据
风险
反证条件
置信度
```

## 9. API 产品化建议

Phase 9 Step 2 已实现 `ResearchModule` 第一版只读 API。这一版刻意只返回 fallback 数据，用于先稳定 Research Center 的产品形态，后续再逐步接入真实行情、持仓、主题强弱、因子信号和复盘历史。

当前接口：

```http
GET /research/playbooks
GET /research/daily-note
GET /research/portfolio-review
GET /research/portfolio-context
GET /research/ideas
GET /research/theses
GET /research/catalysts
```

第一版输出原则：

- 不生成无依据买卖建议
- 所有动作建议必须落入继续持有、分批加仓、观望、止盈、风控等固定分类
- fallback 明确标注为产品骨架，不伪装成真实行情结论
- 后续接入真实数据后，仍保留依据、风险、反证条件和下一步关注

## 10. Dashboard 产品化建议

Research Center 页面建议包含：

```text
顶部：今日一句话结论 + 操作建议
左侧：市场温度 + 主题强弱
中间：持仓体检 + Idea Cards
右侧：Thesis 更新 + 催化剂日历
底部：上次判断 vs 今日验证
```

每张卡片都要有：

- 结论
- 依据
- 风险
- 反证条件
- 下一步动作

## 11. 成功标准

如果 AresQuant 能稳定回答下面问题，就说明产品方向是对的：

1. 今天市场适不适合加仓？
2. 我的持仓有没有明显风险？
3. 哪个主题正在走强，哪个只是在反弹？
4. 哪些标的值得进入观察池？
5. 哪些持仓 thesis 被增强或削弱？
6. 当前策略信号支持我继续持有吗？
7. 我下一步应该继续持有、分批加仓、观望、止盈还是风控？
