# AresQuant 精简用户手册

## 1. 一键启动

在项目根目录执行：

```bash
pnpm dev
```

启动成功后打开：

- 工作台：`http://localhost:5173`
- API 文档：`http://localhost:3000/docs`

如果 `5173` 被占用，终端会自动显示新的前端地址，例如 `http://localhost:5174`。

## 2. 第一次使用前

确保电脑已经打开 Docker Desktop。

如果依赖还没装过，先执行一次：

```bash
pnpm i
```

之后日常只需要：

```bash
pnpm dev
```

## 3. 工作台怎么看

- 总览：看数据、策略、回测和风险状态的总览。
- 数据中心：看股票池、日线行情、财务因子等数据覆盖情况。
- 策略库：看当前可用策略和参数结构。
- 回测：看回测任务、资金、区间和状态。
- 投研中心：看持仓上下文、主题暴露、观察信号和风险标记。
- 风险监控：看当前阶段的边界约束，确认没有进入模拟盘或实盘。

## 4. 当前项目能做什么

- 自动启动本地 Postgres 和 Redis。
- 自动执行 Prisma 生成和数据库迁移。
- 启动 NestJS API 和 React 工作台。
- 写入 Ricki 的组合上下文种子数据。
- 展示数据中心、策略库、回测、投研和风险监控页面。
- 投研中心可以读取真实后端接口返回的持仓上下文。

## 5. 常见问题

### Docker 没启动

先打开 Docker Desktop，再重新执行：

```bash
pnpm dev
```

### 前端端口不是 5173

这是正常的，说明 `5173` 被占用。看终端输出的 `Local` 地址，打开那个地址即可。

### 想停止项目

在运行 `pnpm dev` 的终端里按：

```text
Ctrl + C
```

### 想看接口

打开：

```text
http://localhost:3000/docs
```

## 6. 协作约定

后续新增功能、页面文案、使用说明和需求描述默认使用中文。只有专业名词、代码标识、接口路径、策略 code、技术缩写和必要的英文品牌名保留英文。

## 7. UI 约定

- 桌面端固定导航保留顶部和左侧安全边距，不贴边。
- 页面内容和导航按统一栅格对齐，避免每个模块单独凑位置。
- 操作台页面优先保持信息密度、清晰层级和可扫描性，不做营销式大空白。
- 所有新增可见文案默认中文，专业名词和技术标识按需保留英文。
- 新增组件默认考虑悬浮态、选中态、窄屏布局和文字不溢出。
